### US-001: Lock runtime config contract
Status: in_progress
Depends on: None

Description:
As a developer, I want a typed runtime config contract so the existing pipeline can run from declarative settings instead of hidden code defaults.

Acceptance Criteria:
- [ ] Create configs/alarm_hgt.yaml and alarm_hgt/runtime_config.py as the single source of truth for synthetic generation settings, dataset paths, batching, model hyperparameters, metric Ks, training args, and fixed output directories.
- [ ] If YAML parsing support is not already available in miso, install it with conda run -n miso pip install pyyaml and wire the dependency into the project path used for execution.
- [ ] Example: loading a config that sets n_hid=64, num_layers=4, per_device_train_batch_size=4, outputs.checkpoints_dir=outputs/checkpoints, and outputs.results_dir=outputs/results produces typed config objects with those exact values and no hidden overrides.
- [ ] Negative case: loading a config missing outputs.results_dir, model.num_relations, or smoke split sizes fails fast with a readable validation error before export or training starts.
- [ ] Add focused tests that cover valid config loading and invalid config rejection.
